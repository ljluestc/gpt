# Question
What is Continuous Delivery?

# Category
devops / beginner

# Keywords
continuous delivery, cd, release readiness

# Answer

Continuous Delivery means the system is always in a release-ready state. Code is built, tested, scanned, and validated in environments so it can be safely released to production at any time, but the final step typically keeps manual approval. The difference from Continuous Deployment: Continuous Delivery emphasizes 'always ready to release'; Continuous Deployment emphasizes 'automatically release to production.' Many enterprises choose Continuous Delivery over Continuous Deployment due to compliance, approval, or business rhythm.

# Key Points

- System always release-ready
- Automated pipeline ensures production readiness
- Manual approval before final deployment
- Enables fast and safe releases

# Example

Commit → build → test → deploy to staging → manual approval → production.

# Related Concepts

Continuous Integration, Continuous Deployment
